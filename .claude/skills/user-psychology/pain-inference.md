# User Psychology Skill — 用户心理分析

## 核心框架：痛点冰山模型

```
表面层（用户说的）     → "显卡太热了"
行为层（用户做的）     → 降频、开盖、换散热器
情感层（用户感受的）   → 焦虑、后悔购买、不信任品牌
需求层（用户真正要的） → 安静高性能的使用体验
```

## 隐藏需求推导方法

### 1. 五个为什么（5 Whys）
```
痛点: 显卡驱动崩溃
Why 1: 为什么崩溃？→ 新驱动与游戏不兼容
Why 2: 为什么不兼容？→ 驱动发布前测试不充分
Why 3: 为什么测试不充分？→ 游戏太多无法全覆盖
Why 4: 为什么不能全覆盖？→ 缺乏自动化兼容性测试
Why 5: 用户真正需要什么？→ 驱动更新前的自动兼容性验证机制
```

### 2. JTBD（Jobs to Be Done）
用户不是在"买显卡"，而是在"雇佣显卡完成任务"：
- 功能性任务: 流畅运行游戏/渲染
- 情感性任务: 感觉自己做了明智的购买决策
- 社交性任务: 在朋友面前展示性能/画质

### 3. Kano 模型分类
| 类型 | 说明 | 示例 |
|------|------|------|
| 基本需求 | 没有会不满，有了不会特别满意 | 驱动稳定、不过热 |
| 期望需求 | 做得好满意，做不好不满 | 性能符合标称值 |
| 兴奋需求 | 没有不会不满，有了会惊喜 | AI 自动优化游戏设置 |

## Devil's Advocate（Munger 反向论证）

### 审查标准
对每个隐藏需求推导，Munger 从反面论证：
- **Strong**: 推导逻辑严密，数据支撑充分 → 置信度不变或提升
- **Moderate**: 有一定道理但证据不够 → 置信度降低 10-20%
- **Weak**: 推导有明显漏洞 → 置信度降低 30%+ 或否决

### 常见推导陷阱
1. **过度泛化**: "一个用户说热" → "所有用户都觉得热" ✗
2. **因果倒置**: "买了 5090 的人抱怨功耗" → 不代表功耗是购买障碍
3. **幸存者偏差**: 论坛上抱怨的是少数，沉默的多数可能满意
4. **时间偏差**: 新品发布期抱怨多是正常的，不代表长期趋势
5. **平台偏差**: Reddit 用户偏极客，不代表大众消费者

## 消费者决策心理

### 显卡购买决策因素（按重要性排序）
1. 性价比（价格/性能比）
2. 游戏兼容性（能不能跑我想玩的游戏）
3. 品牌信任（NVIDIA vs AMD 的生态锁定）
4. 散热/噪音（日常使用体验）
5. 功耗（电费 + 电源要求）
6. 未来性（DLSS/FSR、光追、AI 功能）

### 认知偏差在显卡讨论中的表现
- **锚定效应**: "3080 才 699 美元，5080 凭什么 999？"
- **损失厌恶**: 对降频/崩溃的愤怒远大于对性能提升的喜悦
- **从众效应**: "大家都说 A 卡驱动差" → 实际体验可能没那么差
- **确认偏差**: 买了 N 卡的人倾向于找 A 卡的缺点
